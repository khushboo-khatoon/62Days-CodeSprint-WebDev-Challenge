## 🚀 Overview

**CSS Animation Collection** is a high-performance visual dashboard showcasing complex UI components, modern transitions, and continuous loops built almost entirely with vanilla markup stylesheets. 

The environment includes a responsive grid featuring 18 decoupled layout cards. Instead of standard animations, this collection demonstrates advanced styling logic including **3D transforms**, **custom cubic-bezier acceleration mapping**, **clip-path mutations**, **SVG dash-array drawing metrics**, and **matrix stagger loops**—all contained within a live-rendering atmospheric neon scanline CRT filter framework.

---

## 🛠️ Tech Stack

This project is built directly on top of native Web APIs without the overhead of heavy frameworks, preprocessors, or runtime dependencies:

*   **HTML5 Structure:** Accessible semantics utilizing inline custom style properties (`--i`, `--d`) to inject runtime offsets directly into the element tree.
*   **CSS3 Motion Architecture:** Driven entirely by keyframe timelines, absolute positioning layers, hardware-accelerated viewport translations (`perspective`, `backface-visibility`), and high-contrast color filtration loops (`backdrop-filter`).
*   **Vanilla DOM Controls:** Tiny, high-efficiency event listeners (`onclick`) embedded directly into interactive cards to execute native state management toggles dynamically.

## 💻 Cloning & Running Instructions

Because this environment is framework-agnostic, you do not need to download or install package dependencies like `npm` or `yarn`. You can clone it and run it directly inside any secure execution client.

### 1. Clone the Repository
Pull down the project code locally to your environment by running the following command inside your terminal:

```bash
git clone https://github.com/abhisek2004/62Days-CodeSprint-WebDev-Challenge/Project/HTML,CSS,JS/css-animation-collection.git
```

### 2. Navigate to the Directory

Enter the root folder containing the application components:

```bash
cd css-animation-collection
```

### 3. Launch the Application

You can execute the frontend pipeline via multiple access routes depending on your development workflow:

#### Option A: Direct Local Execution (Simplest)

Locate the cloned directory on your computer using your native system browser (Finder or File Explorer) and **double-click** the `index.html` file. It will spin up instantly inside your preferred web browser.

#### Option B: Terminal Python Server

If you want to view it running from a local port layer, run the built-in HTTP server runtime included inside Python:

```bash
# For Python 3.x installations
python -m http.server 8000
```

Once initialized, open your web browser and navigate directly to:

> `http://localhost:8000`

#### Option C: VS Code Live Server Extensions

If you are using Visual Studio Code:

1. Open the project root folder in VS Code.
2. Click the **"Go Live"** toggle button located at the bottom right toolbar, or right-click `index.html` and select **"Open with Live Server"**.
